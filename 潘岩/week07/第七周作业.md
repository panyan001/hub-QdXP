作业一：
从 `model.py` 可以看到，这是一个 联合训练模型 ，两个任务共享 BERT 编码器：
class BertForIntentClassificationAndSlotFilling(nn.Module):
def __init__(self, config):
    super().__init__()
    self.bert = BertModel.from_pretrained(config.bert_dir)  # 共享的BERT编码器
    # 意图识别（文本分类）头
    self.sequence_classification = nn.Sequential(
        nn.Dropout(config.hidden_dropout_prob),
        nn.Linear(config.hidden_size, config.seq_num_labels),  # 768 -> 意图个数
    )
    # 槽位填充（实体识别）头
    self.token_classification = nn.Sequential(
        nn.Dropout(config.hidden_dropout_prob),
        nn.Linear(config.hidden_size, config.token_num_labels),  # 768 -> token类别个数
    )
两个任务的关联：

1. 共享表示学习 ：两个任务共享 BERT 的编码层，意图信息有助于理解句子整体语义，槽位信息有助于捕获局部实体特征
2. 相互增强 ：意图可以提供上下文约束（如"查询公交"意图下，"北京"更可能是出发地而非景点），槽位信息可以帮助细粒度理解句子结构

分别使用的 Loss
从 `main.py` 可以看到：
# 意图识别的loss 实体识别loss
seq_loss = self.criterion(seq_output, seq_label_ids)      # 意图分类 Loss
token_loss = self.criterion(active_logits, active_labels) # 槽位填充 Loss
loss = seq_loss + token_loss
两个任务都使用 nn.CrossEntropyLoss() ：
- 意图识别（seq_loss） ：
  - 输入： seq_output [batch_size, num_intents]
  - 目标： seq_label_ids [batch_size]
  - 本质：多分类问题
- 槽位填充（token_loss） ：
  - 输入： active_logits [有效token数, num_slot_labels]
  - 目标： active_labels [有效token数]
  - 本质：对每个token进行多分类（BIO标注）
多任务训练 Loss = seq_loss + token_loss 的问题及解决方案 
1. 存在的问题
（1）量级不平衡问题

- 意图识别：每个样本只有1个预测（句子级别）
- 槽位填充：每个样本有 seq_len 个预测（token级别，如32个）
- 假设 batch_size=64，一个batch中：
  - 意图预测数：64
  - 槽位预测数：64 × 32 = 2048
- token_loss 的梯度可能主导训练 ，导致意图识别任务学习不足
（2）收敛速度不一致

- 两个任务的难度不同，收敛速度可能差异很大
- 一个任务可能已经收敛，另一个还在学习，简单相加会干扰已收敛任务
（3）任务间竞争

- 两个任务的梯度方向可能冲突，简单相加可能导致"跷跷板"现象（一个任务变好，另一个变差）
2. 解决方案
方案一：加权求和
# 可学习的权重或手动设置
loss = alpha * seq_loss + beta * token_loss

# 动态调整（根据训练阶段）
if epoch < 5:
    loss = 0.3 * seq_loss + 0.7 * token_loss  # 前期侧重槽位填充
else:
    loss = 0.7 * seq_loss + 0.3 * token_loss  # 后期侧重意图识别
方案二：不确定性加权（Uncertainty Weighting）
# 根据任务的不确定性自动调整权重
log_sigma1 = torch.nn.Parameter(torch.zeros(1))
log_sigma2 = torch.nn.Parameter(torch.zeros(1))

loss = torch.exp(-log_sigma1) * seq_loss + log_sigma1 + \
       torch.exp(-log_sigma2) * token_loss + log_sigma2
