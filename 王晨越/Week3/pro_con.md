# Bert
## Pros：
语义理解极强,能处理“我有点冷”这类口语化的意图（自动关联“空调调节”），而不依赖特定关键词.
泛化能力高：即使是用户没说过的表达方式，只要语义接近，BERT 就能识别.
## Cons：
对车载芯片的算力要求高，推理延迟可能比简单模型长.

# Prompt Learning
## Pros:
灵活性：通过改变提示词，可以快速适配新增的意图，无需大规模重新训练整个模型。
## Cons:
工程复杂度：模版（Template）的设计非常考验功力，不当的设计会导致性能大幅下降。

# Regex Rule
## Pros:
绝对精准：对于“导航到...”或“打电话给...”这种格式固定的指令，识别率接近 100%。
零延迟：计算量极小，几乎不占内存，非常适合车载嵌入式环境。
## Cons:
死板：稍微改变语序（如“帮我拨个电话给老王”）就可能失效，导致用户必须重复特定指令。
难以维护：20 个意图如果全用规则，维护成本会随着规则叠加呈指数级增长。

# TF-IDF
## Pros:
训练极快：几秒钟内就能完成 20 个意图的分类训练。
## Cons:
忽略语序：它只看词频。例如“不要播放音乐”和“播放音乐”，TF-IDF 可能都会识别为“媒体控制”，因为它们包含相同的关键词。
