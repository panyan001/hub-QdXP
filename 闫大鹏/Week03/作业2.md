# 作业2：阅读项目计划书  + 初步项目代码，写清楚四个模型的优缺点，形成一个word/markdown提交。

# 关于bert

- **优点**
  - 语义理解力强
  - 能捕捉上下文信息和词语间的关系
  - 能用GPU加速，适合批量处理
  - 内置tokenizer处理文本规范化
- **缺点**
  - 推理速度较慢
  - 对短文本可能会过度拟合
  - 模型体积大，消耗资源多

# 关于 LLM

- **优点**
  - 0样本学习能力
  - 动态提示词
  - 无需训练特定分类模型
  - 使用灵活
-  **缺点**
  - 依赖外部api，有网络延迟
  - 付费
  - 结果可能不稳定

# regex 正则

- **优点**
  - 规则明确，精度高
  - 执行速度快
  - 资源占用少
  - 不依赖外部资源
- **缺点**
  - 只能处理模式固定的文本
  - 难以处理语义变化
  - 维护成本高，需手动更新规则
  - 泛化能力差

# TF-IDF

- **优点**
  - 模型轻量，推理速度快
  - 不依赖GPU资源
  - 对短文本效果好
  - 可解释性较强，可分析特征权重
- **缺点**
  - 无法捕捉语义信息
  - 依赖分词和特征工程质量
  - 需要维护停用词表
  - 对未录入的词处理差
